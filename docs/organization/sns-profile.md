# Sparks Station — SNSプロフィール設計

## 方針

Sparks Stationは「稼げます」ではなく、AI時代の開発者が事例からマネタイズの現実を学ぶメディアとして見せる。

プロフィールで伝えること:

- AI SaaS、個人開発、Micro-SaaSの海外事例を集める。
- 成功だけでなく、売却、撤退、失敗も扱う。
- 日本のエンジニア、個人開発者、技術系起業家が「次に作るもの」「どう売るか」を考えるための保存先にする。

避ける表現:

- 「誰でも稼げる」
- 「最短で成功」
- 「月収○○」
- 過度な煽り、情報商材っぽい表現

## 川上大和の扱い

川上大和は、Sparks Stationの主役ではなく「編集者名」として扱う。表示名に前面表示はしない。Bioまたは固定投稿で軽く出し、完全な匿名まとめアカウントに見えない程度の人格を足す。

メリット:

- 誰が運営しているかが見えて、無機質なまとめアカウントに見えにくい。
- Threadsでは個人の視点がある方が会話されやすい。
- 将来、ニュースレター、コミュニティ、プロダクトに接続しやすい。

注意点:

- 「川上大和が稼ぎ方を教える」構図にしない。
- 成功者アピール、月収訴求、権威づけには使わない。
- Sparks Stationの信頼をメディア名に蓄積する。

## 固定投稿 / 初回投稿

固定投稿では、自己紹介より「このアカウントをフォローする理由」を伝える。

入れる内容:

- AIで開発は簡単になったが、売ることは簡単になっていない。
- Sparks Stationは海外のMicro-SaaS / AI SaaS / 個人開発事例を集める。
- 成功事例だけでなく、失敗、撤退、売却、GTMも扱う。
- 日本のエンジニア、個人開発者、技術系起業家向け。
- 保存して後で見返せる事例アカウントにする。

初回投稿文案:

```text
AIで、作るハードルは一気に下がりました。

でも、売るハードルはむしろ高くなっています。

Sparks Stationでは、
海外のMicro-SaaS、AI SaaS、個人開発、売却、失敗事例を集めて、
日本のエンジニアや個人開発者向けに整理していきます。

「何を作るか」
「どう売るか」
「どこで失敗するか」

その判断材料になる事例を発信します。

後で見返したい事例集として使ってください。
```

川上大和を入れる版:

```text
Sparks Stationは、AI時代の個人開発・AI SaaS・Micro-SaaSの事例を観察するメディアです。

海外で生まれている小さなプロダクト、収益化の工夫、GTM、売却・買収の事例を、日本語でわかりやすく整理していきます。

運営・編集は川上大和というペンネームで行っています。
成功者として語るのではなく、作る人・試す人のための観察メモとして届けます。
```

## 設定メモ

- Instagram Bioは150文字以内に収める。
- Threadsはプロフィールを個別編集できる。Instagramからインポートもできるが、Sparks StationではThreadsだけ少し会話寄りのBioにしてよい。
- ユーザー名は30文字以内を目安に、英数字、ピリオド、アンダースコアで構成する。

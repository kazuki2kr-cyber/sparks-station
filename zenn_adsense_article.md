## はじめに

個人開発で運営しているポータルサイト **[Sparks Station](https://sparks-station.com)** が、Google AdSenseの審査で **「有用性の低いコンテンツ」** を理由に不承認となりました。

![AdSense不承認画面](/images/adsense-rejection.png)
*実際の不承認画面。ポリシー違反として「有用性の低いコンテンツ」が指摘された。*

サイトの記事自体は1本あたり5,000〜8,000字の独自分析記事を18本以上公開しており、コンテンツの質には自信がありました。にもかかわらず審査に落ちた原因を調査し、**ある戦略的な修正を行ったところ、再申請で一発合格**しました。

この記事では、その原因分析から具体的な修正内容、そして合格までのプロセスを共有します。

## サイト構成

まず、審査落ちした時点でのサイト構成を説明します。

**Sparks Station** は Next.js (App Router) + Firebase Hosting で構築されたポータルサイトで、**2つの性質が異なるコンテンツ**が1つのドメインに同居していました。

```
sparks-station.com/
├── /                      ← ブログ（Portal）トップ
├── /posts/[slug]          ← 記事ページ（18記事）
├── /categories/[slug]     ← カテゴリ一覧
├── /about                 ← 運営者情報
├── /privacy               ← プライバシーポリシー
├── /contact               ← お問い合わせ
├── /tools                 ← おすすめツール紹介
└── /FantasyQuizzesKingdom/ ← クイズゲームアプリ（Web App）
    ├── /solo
    ├── /room/[id]
    ├── /host/[id]
    └── ...
```

### ブログ部分（Portal）
海外SaaSの成功・失敗事例を独自の分析フレームワークで深掘りする記事サイト。1記事あたり5,000〜8,000字で、以下のような構成を持つオリジナルコンテンツです。

- ① サマリー（30秒でわかる概要）
- ② Fact（サービス詳細とTech Stack）
- ③ Insight（なぜ成功/失敗したのか？構造分析）
- ④ Verification（仮説検証）
- ⑤ Localize（日本市場への転用アイデア）

### クイズアプリ部分（Fantasy Quizzes Kingdom）
リアルタイム対戦型のWebクイズアプリ。Firebaseのリアルタイムデータベースを使った本格的なアプリケーションです。忘年会やパーティなどの余興で使えるクイズゲームとして開発しました。

## 原因分析：なぜ「有用性の低いコンテンツ」と判定されたのか？

審査落ちの通知を受けて、サイト全体を精査しました。

ブログ記事自体は独自性が高く、質・量ともに十分だと判断できました。では何が問題だったのか？

### 仮説：クイズアプリがサイト全体の評価を下げている

AdSenseのクローラーは、**ドメイン全体**を評価対象とします。つまり、`sparks-station.com` 配下のすべてのページが審査の対象です。

ここで問題になったのが、クイズアプリ部分 (`/FantasyQuizzesKingdom/`) の特徴です。

| 観点 | ブログ部分 | クイズアプリ部分 |
|------|-----------|---------------|
| テキスト量 | 5,000〜8,000字/ページ | ほぼなし（UIのみ） |
| レンダリング | SSG/SSR（サーバーサイド） | CSR（クライアントサイド） |
| クローラーから見えるコンテンツ | 豊富 | ほぼ空 |
| ページ数 | 18記事 + 固定ページ | 多数の動的ページ |

**クイズアプリはReactのクライアントサイドレンダリング（CSR）で構築されているため、クローラーがアクセスした時点ではほぼ空のHTMLしか存在しません。** さらに、ゲームのルーム画面やホスト画面など、認証後にしかコンテンツが表示されない動的ページが多数あります。

つまりAdSenseのクローラーから見ると、このサイトは「中身のあるページ」と「中身のないページ」が混在しており、後者が全体の評価を引き下げていた可能性が高いと判断しました。

## 実施した対策

### 戦略：「ドメインの承認」を最優先する

AdSenseの審査は**ドメイン単位**で行われます。一度ドメインが承認されれば、その後は個別ページへの広告掲載が可能になります。

そこで、以下の戦略を立てました。

> **審査中は、クイズアプリ部分をクローラーから完全に隠す。**
> **ブログ記事の品質だけで審査に通す。**
> **合格後に、クイズアプリ部分のブロックを解除する。**

### 修正1：`robots.txt` でクローラーをブロック

Next.jsの `robots.ts` を修正し、クイズアプリ部分全体へのクローラーアクセスをブロックしました。

```diff:src/app/robots.ts
 disallow: [
-    '/FantasyQuizzesKingdom/admin',
-    '/FantasyQuizzesKingdom/room/',
-    '/FantasyQuizzesKingdom/host/',
-    '/FantasyQuizzesKingdom/debug',
-    '/FantasyQuizzesKingdom/design-preview',
+    '/FantasyQuizzesKingdom/',
 ],
```

元々は管理画面やルーム画面など一部のパスだけをブロックしていましたが、**ディレクトリごとまるっとブロック**するように変更しました。

### 修正2：`sitemap.xml` からクイズアプリを除外

`sitemap.ts` から、クイズアプリ関連のURLをすべて削除しました。

```diff:src/app/sitemap.ts
  ...postUrls,
- // Quiz App URLs
- {
-     url: `${baseUrl}/FantasyQuizzesKingdom`,
-     ...
- },
- {
-     url: `${baseUrl}/FantasyQuizzesKingdom/solo`,
-     ...
- },
- // ... 他のFQK関連URL
```

`robots.txt` でブロックしていても、サイトマップに含まれているとクローラーが混乱する可能性があるため、整合性を取りました。

### 修正3：About ページの運営者情報を拡充（E-E-A-T対策）

AdSense審査では、サイトの信頼性（E-E-A-T: Experience, Expertise, Authoritativeness, Trustworthiness）も重視されます。

About ページの運営者情報を、より具体的に拡充しました。

```diff:src/app/(portal)/about/page.tsx
  <dl className="grid grid-cols-[100px_1fr] gap-y-2
+     border-l-2 border-emerald-500/20 pl-4">
      <dt>名称</dt>
      <dd>Sparks Station 運営事務局</dd>
+     <dt>開設</dt>
+     <dd>2026年1月</dd>
+     <dt>事業内容</dt>
+     <dd>SaaSビジネスの事例研究・情報発信、
+         Webアプリケーション開発</dd>
      <dt>URL</dt>
      <dd>https://sparks-station.com</dd>
```

## 再申請から合格まで

修正をデプロイした後、以下の手順で再申請しました。

1. **Google Search Console** でサイトマップを再送信
2. 数日待ってからAdSense管理画面で「問題を修正しました」にチェック
3. 「審査をリクエスト」をクリック

結果、**再申請で無事にAdSense審査を通過**しました 🎉

## 合格後の対応

ドメインの承認が得られた後は、`robots.txt` と `sitemap.xml` を元に戻し、クイズアプリ部分も通常通りクローラーがアクセスできるようにしています。

これにより、クイズアプリ部分への自然検索流入も回復し、将来的にはアプリ内にも広告を掲載できる状態になりました。

## まとめと教訓

### 1. 「1ドメインに異なる性質のコンテンツが同居する」リスク

ブログとWebアプリを同一ドメインで運用する場合、**テキストコンテンツの薄いアプリ部分が、サイト全体の評価を引き下げるリスク**があります。特にAdSense審査のような「ドメイン単位の評価」では、これが致命的になり得ます。

### 2. 「まず通す、その後広げる」戦略

審査に通すことが最優先であれば、**評価の足を引っ張るコンテンツを一時的に隠し、高品質な部分だけで勝負する**のは有効な戦略です。合格後に隠していた部分を開放すれば、最終的にはどちらにも広告を掲載できます。

### 3. robots.txt と sitemap.xml の整合性

`robots.txt` でブロックしていても `sitemap.xml` に含まれているとクローラーが混乱します。**両方を同時に修正し、整合性を保つ**ことが重要です。

### 4. E-E-A-T は基本だが大事

運営者情報、プライバシーポリシー、お問い合わせページなど、**サイトの信頼性を示す基本的なページ**が整っていることは前提条件です。これらが不足している場合は、コンテンツの質以前に落ちる可能性があります。
